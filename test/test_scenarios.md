# 테트리스 게임 테스트 시나리오

## 1. 개요
PRD(제품 요구사항 문서)와 1:1 매핑된 레트로 테트리스 게임의 테스트 시나리오입니다.

## 2. 테스트 시나리오

| 테스트 ID | 요구사항 섹션 | 요구사항 설명 | 테스트 케이스 | 예상 결과 |
| :--- | :--- | :--- | :--- | :--- |
| **TID-001** | 2.1 테트로미노 | 총 7종류의 테트로미노 (I, O, T, S, Z, J, L)와 특정 색상 | 여러 게임을 진행하며 생성되는 블록 관찰. | 7가지 모양(I-청록, O-노랑, T-보라, S-녹색, Z-빨강, J-파랑, L-주황)이 모두 올바른 색상으로 나타남. |
| **TID-002** | 2.2 게임 플레이 | 블록 무작위 생성 | 게임을 플레이하고 블록의 순서 기록. | 블록이 반복되는 고정 패턴이 아닌 무작위 순서로 나타남. |
| **TID-003** | 2.2 게임 플레이 | 블록 이동 및 회전 | 왼쪽/오른쪽 화살표 키와 위쪽 화살표 키 입력. | 블록이 좌/우로 이동하고 시계 방향으로 90도 회전함. |
| **TID-004** | 2.2 게임 플레이 | 자동 낙하 | 게임 시작 후 아무 키도 누르지 않음. | 설정된 간격으로 블록이 자동으로 한 칸씩 아래로 이동함. |
| **TID-005** | 2.2 게임 플레이 | 소프트 드롭 | 아래쪽 화살표 키 입력. | 블록이 자동 속도보다 빠르게 떨어짐. |
| **TID-006** | 2.2 게임 플레이 | 하드 드롭 | 스페이스바 입력. | 블록이 즉시 가장 낮은 위치로 떨어져 고정됨. |
| **TID-007** | 2.2 게임 플레이 | 충돌 감지 | 블록을 벽이나 기존 블록 쪽으로 이동. | 블록이 벽(최소/최대 X)에서 멈추고 기존 블록이나 바닥(최대 Y)을 통과하지 못함. |
| **TID-008** | 2.2 게임 플레이 | 줄 삭제 | 가로 한 줄을 블록으로 완전히 채움. | 채워진 줄이 사라지고 그 위의 블록들이 한 줄씩 아래로 이동함. |
| **TID-009** | 2.2 게임 플레이 | 점수 계산 | 한 번에 1, 2, 3 또는 4줄 삭제. | 삭제된 줄 수에 따라 점수가 증가함. |
| **TID-010** | 3. 조작 | 왼쪽 화살표 키 | 왼쪽 화살표 키 누름. | 활성 블록이 왼쪽으로 한 칸 이동. |
| **TID-011** | 3. 조작 | 오른쪽 화살표 키 | 오른쪽 화살표 키 누름. | 활성 블록이 오른쪽으로 한 칸 이동. |
| **TID-012** | 3. 조작 | 위쪽 화살표 키 | 위쪽 화살표 키 누름. | 활성 블록이 시계 방향 회전. |
| **TID-013** | 3. 조작 | 아래쪽 화살표 키 | 아래쪽 화살표 키 누름. | 활성 블록이 소프트 드롭 수행. |
| **TID-014** | 3. 조작 | 스페이스바 | 스페이스바 누름. | 활성 블록이 하드 드롭 수행. |
| **TID-015** | 4.1 메인 게임 보드 | 그리드 크기 10x20 | 특정 그리드 치수. | 상호작용 영역이 정확히 가로 10칸, 세로 20칸임. |
| **TID-016** | 4.2 HUD | 점수 표시 | 점수 UI 요소 확인. | 줄이 삭제될 때마다 점수가 실시간으로 업데이트됨. |
| **TID-017** | 4.2 HUD | 다음 블록 표시 | 다음 블록 UI 요소 확인. | 다음에 생성될 블록의 모양을 표시함. |
| **TID-018** | 4.2 HUD | 게임 오버 조건 | 블록을 맨 위(0행)까지 쌓음. | 게임이 중단되고 "Game Over" 메시지와 최종 점수가 표시됨. |
