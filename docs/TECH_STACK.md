# 테트리스 게임 기술 스택 선정 문서 (TECH_STACK)

## 1. 개요
본 문서는 웹 브라우저 기반 테트리스 게임 개발을 위한 기술 스택 선정 내역과 선정 근거, 호환성 검증 결과를 기술합니다. 초보 개발자도 쉽게 접근 가능하며, 향후 유지보수와 확장에 용이한 기술을 중심으로 선정하였습니다.

## 2. 선정 기술 스택 (Tech Stack)

### 2.1. 프론트엔드 (Frontend) & 빌드 도구
*   **Core**: HTML5, CSS3, JavaScript (ES6+)
    *   **선정 근거**: 웹 표준 기술로, 별도의 복잡한 학습 없이 바로 게임 로직 구현에 집중할 수 있습니다.
*   **Rendering**: HTML5 Canvas API
    *   **선정 근거**: 2D 그래픽 처리에 최적화되어 있으며, 테트리스와 같은 격자 기반 게임 구현에 매우 효율적입니다. DOM 조작 방식보다 성능이 우수합니다.
*   **Build Tool**: **Vite** (무료 / 오픈소스)
    *   **선정 근거**: 빠르고 설정이 간편한 차세대 프론트엔드 빌드 도구입니다. `create-react-app` 등보다 가볍고 빠르며, 최신 모던 웹 개발 생태계의 표준으로 자리 잡고 있습니다. 향후 TypeScript나 React 등으로 확장하기도 용이합니다.

### 2.2. 개발 환경 (IDE & Tools)
*   **IDE**: **Visual Studio Code (VS Code)** (무료)
    *   **선정 근거**: 전 세계적으로 가장 많이 사용되는 무료 코드 에디터입니다. 방대한 확장 프로그램 생태계(Prettier, ESLint 등)를 지원하여 생산성을 높여줍니다.
*   **Version Control**: **Git** (무료)
    *   **선정 근거**: 소스 코드 형상 관리를 위한 산업 표준입니다.

## 3. 호환성 및 공식 문서 검증 결과

### 3.1. Vite (Build Tool)
> **검증 대상**: [Vite 공식 문서 (Getting Started)](https://vitejs.dev/guide/)
*   **설치 가이드 확인**: `npm create vite@latest` 명령어를 통해 최신 버전의 프로젝트를 손쉽게 생성할 수 있음을 확인했습니다.
*   **요구 사항**: Node.js 18+ 또는 20+ 버전을 요구합니다. (최신 LTS 버전 사용 권장)
*   **호환성**: 최신 모던 브라우저(Chrome, Edge, Firefox 등) 환경에서 ES 모듈을 기반으로 최적화된 개발 경험을 제공합니다.

### 3.2. HTML5 Canvas
> **검증 대상**: [MDN Web Docs (Canvas API)](https://developer.mozilla.org/en-US/docs/Web/API/Canvas_API)
*   **호환성**: 모든 최신 데스크톱 및 모바일 브라우저에서 별도의 플러그인 없이 기본 지원됩니다.

## 4. 추천 소프트웨어 (유료/무료 구분)

| 구분 | 소프트웨어 (도구) | 가격 | 용도 | 비고 |
| :--- | :--- | :--- | :--- | :--- |
| **필수** | **VS Code** | **무료** | 코드 작성 및 편집 | 추천 확장: Live Server (Vite 사용 시 불필요하지만 참고), Prettier |
| **필수** | **Google Chrome** | **무료** | 게임 실행 및 디버깅 | 개발자 도구(F12) 활용 |
| **필수** | **Node.js** | **무료** | 자바스크립트 런타임 | Vite 실행을 위해 필요 (LTS 버전 권장) |
| 선택 | GitHub Copilot | 유료 | AI 코딩 보조 | 코드 자동 완성 및 제안 (생산성 향상) |
| 선택 | WebStorm | 유료 | 통합 개발 환경 (IDE) | VS Code보다 강력한 기능을 제공하나 유료임 |

## 5. 결론 및 제안
가장 **심플하고 강력한** 조합인 **Vite + Vanilla JS + Canvas API** 조합으로 진행하는 것을 권장합니다.
이 조합은 초기 설정이 거의 필요 없고, 초보자가 구조를 이해하기 쉬우며, 나중에 React나 TypeScript 등을 도입하고 싶을 때도 Vite 환경 그대로 확장이 가능합니다.
